Contribute

Contributions to Savime are welcome. Before making a contribution, follow the steps below listed:
  1. Check if the code conventions agree with the ones already used on the system. SAVIME doesn't yet have a formal coding standard, but a few conventions have been used in its development, so try to follow them when possible. A few simple examples include:
    • Use of capital letters to delimit multiple words in functions and class names
    • In code file names, use of lowercase and underscores to delimit multiple words
    • Do not break line for curly brackets in loops, conditionals and function definitions
  2. Check if your contribution is well documented
  3. Check if your code works well with the latest Savime update
  4. Check if your contribution breaks any of Savime automated tests
  5. Do the pull request. Check Github documentation
  6. If your code has any new dependencies, inform these on the pull request
  7. Follow the progress of your request
